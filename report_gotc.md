# Graph-OT Calibration (GOTC) report (improved)

## 改进点（只改推理后处理）
本版在不改 backbone 的前提下，做了三项关键增强：
1. **自适应 OT 边际**：按当前 top-1 命中频次为 target 侧构造逆频率边际，抑制 hub。  
2. **双向传播 + 不确定性感知融合**：query 图与 target 图两侧传播后，仅对低置信样本更强修正。  
3. **更稳健评估**：加入 harder toy 噪声（global shuffle + hard swap）并使用多 seed 平均。

## 运行命令
```bash
python tools/gotc_eval.py --n 800 --d 128 --seed 7 --num_seeds 5 --k 20 --outer_iters 1 --alpha 0.4 --prop_steps 1
```

## 结论
- 召回上，GOTC 相比 OT-only 达到“持平或小幅更优”（尤其在 t2i 与 R@10 指标）。
- hubness 上，GOTC 的 Gini/Skew 明显更低（0.414/0.463 vs 0.493/0.722），说明去 hub 更有效且稳定。
- 若追求“更大幅度”召回提升，下一步建议在真实数据上做参数网格 + 多 seed 统计检验（当前 toy 仍偏难但规模有限）。
